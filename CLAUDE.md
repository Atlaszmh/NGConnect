## Project Overview

## Key Architecture Decisions

### .env Loading
There is **no `dotenv` package**. The `.env` file at the project root is loaded via Node.js 20's native `--env-file` flag:
- **Production:** `node --env-file=../.env dist/index.js` (server runs from `server/` dir)
- **Dev:** `NODE_OPTIONS=--env-file=../.env tsx watch src/index.ts`
- **Windows Service:** `--env-file=C:\Users\Dragon\Documents\Projects\NGConnect\.env` passed as a nodeOption in `service.ts`

Do NOT add dotenv as a dependency.

### Express 5 + path-to-regexp
Express 5 uses path-to-regexp v8 which requires named wildcard params. Use `/*path` not `/*`:
```typescript
// WRONG (throws PathError in Express 5)
router.all('/*', handler)
app.get('*', handler)

// CORRECT
router.all('/*path', handler)
app.get('/*path', handler)
```
This fix is applied in: `server/src/routes/sonarr.ts`, `server/src/routes/radarr.ts`, `server/src/index.ts`.

### Helmet Config
Helmet's CSP `upgrade-insecure-requests` directive causes the browser to upgrade HTTP→HTTPS, breaking LAN access. Helmet 8 does not support `null` to remove individual directives. The fix is:
```typescript
app.use(helmet({
  contentSecurityPolicy: false,
  hsts: false,
}));
```

### Static File Serving
The `if (process.env.NODE_ENV === 'production')` block in `index.ts` serves the frontend. This only activates when `NODE_ENV=production` — ensured by `cross-env` in the `start` script. In dev mode, the Vite dev server on `:5173` handles the frontend.

### Production Build
```bash
npm run build     # from project root — runs tsc in server/ and vite build in client/
npm start         # serves frontend + API from :3001
```

## Windows Service Setup

NGConnect is installed as a Windows service via [node-windows](https://github.com/coreybutler/node-windows).

**Install/uninstall:**
```bash
cd server
npm run service:install    # requires elevated terminal
npm run service:uninstall
```

**Service config file:** `server/src/daemon/ngconnect.xml`
- This XML is generated by node-windows and controls the actual Windows service
- If `service.ts` changes paths/args but the XML already exists, node-windows may use the cached XML
- In that case, edit `ngconnect.xml` directly and restart the service via `services.msc`

**Key service settings in the XML:**
- Executable: `C:\Program Files\nodejs\node.exe`
- Args include: `--env-file=C:\Users\Dragon\Documents\Projects\NGConnect\.env` and `--file C:\Users\Dragon\Documents\Projects\NGConnect\server\dist\index.js`
- Env: `NODE_ENV=production`
- Account: `LocalSystem`

## Full Windows Service Stack

All four services auto-start on Windows boot:

| Service | Port | Type |
|---|---|---|
| Radarr | 7878 | Built-in Windows service |
| Sonarr | 8989 | Installed via `ServiceInstall.exe` in Sonarr dir |
| SABnzbd | 8080 | Installed via `SABnzbd-console.exe -f <ini> install` |
| NGConnect | 3001 | node-windows (`npm run service:install`) |

**Sonarr service note:** Sonarr v4 uses `ServiceInstall.exe` (in the Sonarr install dir). The old `-install` CLI flag is v3 only. Sonarr service requires `LocalSystem` account for data directory access.

## LAN Access

- Server binds to `0.0.0.0:3001` — accessible from any local device
- Windows Firewall rule required (one-time, run as admin):
  ```powershell
  New-NetFirewallRule -DisplayName 'NGConnect' -Direction Inbound -Protocol TCP -LocalPort 3001 -Action Allow -Profile Any
  ```
- Helmet HSTS is disabled (see above) to prevent HTTPS enforcement on HTTP LAN

## Common Debugging

**Check which process is on port 3001:**
```powershell
(Get-NetTCPConnection -LocalPort 3001 -State Listen).OwningProcess | ForEach-Object { (Get-Process -Id $_).CommandLine }
```

**View server logs:**
```
server/logs/ngconnect-YYYY-MM-DD.log
server/logs/error-YYYY-MM-DD.log
```

**Service not starting after XML edit:**
1. Open `services.msc`
2. Find `NGConnect`
3. Right-click > Restart

**Dev server vs production confusion:**
- Dev: Vite HMR on `:5173`, API on `:3001` via `tsx watch`
- Production: Everything on `:3001`, `NODE_ENV=production` required
- The static file serving block in `index.ts` is gated on `NODE_ENV === 'production'`
